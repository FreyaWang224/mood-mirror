Project Overview

AI 情绪日记

这是一个面向学生和年轻人的 AI 情绪记录原型。用户先写下今日片段，系统再生成情绪分析、真实诗句和可视化共鸣卡，让日记从“记录”变成“被理解”。

Prototype Type单页 HTML 可交互原型
Core Experience记录 → 分析 → 共鸣卡
Current StageMVP / 课堂展示版

1. 想法与定位

产品想法

用户写下今天发生的事，AI 帮助整理情绪，并用一句真实诗词和一张视觉意象图概括当天状态。

目标用户

学生、年轻上班族，以及想记录情绪但不想写长篇日记的人。

产品定位

不是心理诊断工具，而是轻量的情绪陪伴与自我观察工具。

第一版目标

验证用户是否能顺利完成“写下心情 → 获得反馈 → 保存记录”的核心路径。

情绪记录 AI 分析 真实诗词 视觉共鸣卡

2. 用户流程

1

进入页面

看到产品名称、流程提示和今日记录入口。

2

选择心情

从开心、平静、焦虑、难过、疲惫、生气中选择一种。

3

写下片段

输入今天发生的事情或情绪原因。

4

生成结果

系统展示 AI 分析、真实诗句和视觉意象卡。

5

保存日记

用户确认结果后保存到历史记录。

3. 业务模块拆解

Module 1: App Header

展示产品名称、定位和当前原型目标，让用户理解这是一个情绪记录工具。

Module 2: Mood Selector

提供六种基础情绪选项，作为后续分析、诗句匹配和图片切换的输入。

Module 3: Diary Input

让用户写下今天的事件或感受，是 AI 分析的主要上下文。

Module 4: AI Emotion Analysis

模拟生成情绪总结、可能原因、温柔建议和关键词。

Module 5: Resonance Card

将分析结果转化成真实诗句、情绪意象和 AI 生成图片，形成产品亮点。

Module 6: Diary History

使用浏览器本地存储保存最近记录，用来展示闭环体验。

Module 7: Prototype Comments

记录当前原型的设计判断、限制和下一步迭代方向。

4. AI 参与点

情绪分析

根据用户选择的心情和日记文本，输出情绪总结、可能原因和温柔建议。

诗句匹配

当前版本使用公版古诗词库，按情绪匹配真实诗句，降低纯 AI 句子的生硬感。

视觉意象

当前版本使用一张 AI 生成的六情绪图集，按心情裁切显示对应画面。

5. MVP 范围

第一版包含

情绪选择、日记输入、结果生成、共鸣卡展示、历史记录、原型评论。

第一版暂不包含

登录注册、真实后端数据库、真实 AI API、心理诊断、用户社区、复杂数据可视化。

6. Prototype Comments

Comment 1

原型已从“左右同时展示”调整为“先记录、再生成结果”的真实使用流程，更接近最终产品体验。

Comment 2

真实诗词提供文化质感，避免结果过于模板化或人机感过重。

Comment 3

AI 生成图片作为视觉共鸣卡的主体，但当前仍是固定图集裁切，不是每篇日记实时生成。

Comment 4

下一版可以接入真实模型，让分析内容、诗句匹配和视觉意象更贴合具体文本。
