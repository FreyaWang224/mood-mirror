# 情绪星球交互原型设计

## 1. 目标

将现有的 AI 情绪日记静态页面升级为一个具有角色陪伴感的沉浸式交互原型。

原型借鉴陪伴类应用中“角色与用户对话、场景承载功能”的交互形式，但不复制任何现有产品的具体角色、布局或视觉资产。角色设计为原创金发星际旅人，整体世界观采用“主星球 + 每日共鸣星球”。

本轮仍以验证产品体验为目标，不接入真实 AI 服务。现有的模拟情绪分析、真实诗句匹配、视觉意象和本地历史记录继续保留。

## 2. 体验原则

- 角色是用户进入功能的情感入口，而不是单纯装饰。
- 场景变化承担流程变化，减少传统网页面板堆叠感。
- 每次记录都生成一颗独特的共鸣星球，让日记留下可见痕迹。
- 情感回应先于完整分析，避免结果像冷冰冰的数据报告。
- 动画服务于叙事，不牺牲可读性、操作效率和设备性能。
- 手机与电脑功能一致，手机强调沉浸，电脑扩展景观。

## 3. 视觉方向

采用“电影感 3D 场景 + 水彩纸张 UI”的混合语言：

- 星空、星球和角色使用精细的风格化 3D 质感。
- 光影以深靛蓝夜空、暖金色轮廓光、玫瑰红和苔藓绿为主。
- 对话框、输入区域和共鸣卡采用轻薄水彩纸张质感。
- UI 浮层尽量轻盈，不使用大面积不透明白色卡片遮挡场景。
- 场景必须是首屏主体，避免营销式首屏和普通仪表盘布局。
- 角色为原创金发星际旅人，不直接复刻《小王子》或其他现有 IP。

## 4. 世界结构

### 4.1 主星球

主星球是用户长期返回的家园，承载：

- 原创旅人的待机与陪伴互动
- 今日记录入口
- 近期日记形成的玫瑰花园
- 进入完整历史星图的天文台

### 4.2 共鸣星球

每次完成日记后，系统根据当日情绪展示一颗新的共鸣星球。星球的色彩、天气、植物、光线和物件随情绪类型变化。

共鸣星球承载：

- 角色的一句核心情感回应
- 情绪摘要与原因
- 温和建议
- 真实诗句及出处
- 当日视觉意象
- 保存和返回主星球操作

### 4.3 历史宇宙

- 最近的记录以花朵形式生长在主星球玫瑰花园中。
- 完整历史以星空图鉴展示，每篇日记对应一颗共鸣星球。
- 点击历史星球可重新进入当天的共鸣场景并查看记录。

## 5. 核心流程

### 5.1 今日记录

1. 用户进入主星球，角色处于轻微待机状态。
2. 用户点击角色。
3. 角色询问：“今天有什么事情留在你心里？”
4. 对话后展开水彩纸张式记录界面。
5. 用户选择情绪并填写日记。
6. 用户点击“带我去看看今天的心情”。

角色对话负责建立陪伴感，完整输入框负责承载较长文字，两者不能互相替代。

### 5.2 星际转场

1. 记录界面收起。
2. 角色跑向星球边缘。
3. 角色乘流星飞离主星球。
4. 镜头穿过星云并抵达当日共鸣星球。

标准转场约 2 至 3 秒，并提供“跳过”按钮。开启系统“减少动态效果”时，改用短淡入淡出。

### 5.3 AI 分析与共鸣卡

1. 抵达共鸣星球后，角色先说一句核心感受。
2. 用户继续后，完整共鸣卡展开。
3. 共鸣卡展示分析、诗句、建议与视觉意象。
4. 用户保存日记。
5. 共鸣星球进入历史星图，主星球花园新增一朵花。
6. 用户返回主星球。

## 6. 页面模块

原型保留单个 HTML 入口，内部划分为五个职责清晰的模块：

### 6.1 主星球场景模块

负责背景、角色待机动画、花园入口、天文台入口和场景热点。

### 6.2 日记对话模块

负责角色对白、情绪选择、日记输入、字数状态和提交校验。

### 6.3 星际转场模块

负责奔跑、流星轨迹、镜头推进、跳过操作和减少动态效果适配。

### 6.4 共鸣星球模块

负责情绪主题场景、核心回应、完整共鸣卡、保存与返回操作。

### 6.5 历史星图模块

负责近期花园、全部历史记录、历史星球浏览和旧日记回访。

模块通过统一的应用状态切换，不依赖页面刷新。

## 7. 视觉素材拆分

视觉不能全部烘焙在一张图中。计划生成并独立使用：

- 主星球无文字背景
- 原创旅人待机、对话、奔跑和旅行状态素材
- 不同情绪主题的共鸣星球背景
- 星空、星云和流星转场素材
- 水彩纸张纹理
- 花园、天文台及历史星球所需的小型场景素材

HTML、CSS 和 JavaScript 负责文字、控件、响应式布局、动画时序和交互状态，保证内容清晰且可维护。

## 8. 数据与状态

本轮继续使用浏览器 `localStorage`，不接入真实后端。

主要状态：

- `home-idle`：主星球待机
- `journal-dialogue`：角色开始对话
- `journal-editing`：情绪与日记输入
- `journey-loading`：准备生成结果
- `journey-traveling`：星际转场
- `resonance-intro`：角色核心回应
- `resonance-card`：完整分析展开
- `history-map`：历史星图

每篇记录至少保存日期、原文、所选情绪、分析结果、诗句、视觉意象和对应星球主题。

## 9. 异常与边界状态

- 日记为空时禁止出发，由角色给出温和提醒。
- 生成期间禁止重复提交。
- 转场可跳过，跳过后直接进入共鸣星球。
- 保存成功后才更新花园和历史星图。
- 没有历史记录时，花园保持初始状态，天文台显示空星图提示。
- 图片加载失败时使用场景底色和简化装饰，核心功能仍可操作。
- 低性能设备和“减少动态效果”环境自动使用简化动画。

## 10. 响应式策略

### 手机端

- 竖屏全屏构图，角色和场景占据主要区域。
- 水彩输入层从底部展开。
- 操作按钮位于拇指容易触达的位置。
- 共鸣卡采用纵向阅读。

### 电脑端

- 横屏扩展完整主星球景观。
- 角色与互动地点形成空间关系。
- 输入层和共鸣卡出现在不遮挡主体的侧下方。
- 不将手机画面简单拉伸或放进模拟手机边框。

## 11. 动画范围

本轮采用“场景动画 + 电影式转场”，不开发真正的可自由控制 3D 游戏：

- 角色具有呼吸、观察、转身和短距离奔跑动画。
- 星空具有缓慢粒子漂浮和微光变化。
- 主星球到共鸣星球使用镜头推进、流星轨迹和场景淡入。
- 动画由 CSS 和 JavaScript 状态控制。
- 不引入角色自由行走、物理碰撞或复杂 3D 操作。

## 12. 验收标准

- 用户能完整走通“主星球记录 → 星际旅行 → 共鸣星球 → 保存 → 历史回访”。
- 角色是功能入口，并在关键节点提供情感回应。
- 手机和电脑均无文字溢出、图片拉伸、内容遮挡或明显布局跳动。
- 转场时长合理且可跳过。
- 刷新页面后，已保存的历史记录仍然存在。
- 花园和星图能正确反映保存后的记录。
- 所有控件具有清晰的可交互状态。
- 开启“减少动态效果”后仍可完整使用。
- 视觉达到已确认的“电影感 3D + 水彩 UI”方向，同时保持原创。

## 13. 本轮不包含

- 真实 AI API 调用
- 用户账户和跨设备同步
- 云端数据库
- 实时 AI 图片生成
- 完整 3D 模型和自由移动
- 多人社交或公开分享

这些能力应在原型流程和视觉方向验证完成后，按独立模块逐步加入。
